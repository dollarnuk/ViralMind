# Журнал виправлення помилок (ERRORS.md)

Цей файл містить опис проблем, які виникли під час розробки ViralMind, та методи їх вирішення. Це допоможе наступним розробникам (або ШІ) зрозуміти історію змін.

---

### 📅 2026-02-25

#### 1. YouTube Bot Detection (Блокування YouTube)
*   **Симптом**: Помилка `Sign in to confirm you’re not a bot` при спробі завантажити відео через `yt-dlp`.
*   **Причина**: YouTube ідентифікує IP-адресу VPS (Contabo) як бота.
*   **Спроба 1**: Додавання розширених заголовків (headers) та імітація реальних браузерів (User-Agent).
    *   *Результат*: Не допомогло, Contabo IP все одно в чорному списку.
*   **Спроба 2**: Додавання підтримки `cookies.txt` та монтування цього файлу в Docker-контейнери.
    *   *Результат*: **Працює**, якщо користувач завантажить власні кукі з браузера.

#### 2. MediaPipe Attribute Error (Помилка ШІ)
*   **Симптом**: `AttributeError: module 'mediapipe' has no attribute 'solutions'` у сервісі кадрування.
*   **Причина**: Некоректний імпорт у специфічному середовищі Docker (slim-python) або відсутність системних бібліотек.
*   **Рішення**:
    1.  Змінено стиль імпорту на прямий: `import mediapipe.solutions.face_detection as mp_face_detection`.
    2.  Додано `libglib2.0-0` у `Dockerfile`.
    *   *Результат*: **Виправлено**.

#### 3. Celery Task Registration (Завдання не знайдено)
*   **Симптом**: `KeyError: 'process_video'` при спробі запустити обробку.
*   **Причина**: Celery не бачив файл `worker.py`, оскільки відсутні файли `__init__.py` і він не був прописаний у параметрі `include`.
*   **Рішення**: Створено порожні `__init__.py` у всіх папках та оновлено конструктор Celery: `include=['app.tasks.worker']`.
    *   *Результат*: **Виправлено**.

#### 4. Malformed Error Response (Невідома помилка в UI)
*   **Симптом**: При помилці в Celery на сайті відображалося "Невідома помилка", хоча в логах вона була.
*   **Причина**: API намагався отримати статус завдання, але коли завдання падало, Celery повертав об'єкт виключення (Exception), який неможливо було передати через JSON.
*   **Рішення**: У ендпоінті `task-status` додано блоку `try/except` та примусову конвертацію будь-якої помилки в рядок (string).
    *   *Результат*: **Виправлено**. Тепер UI показує текст помилки.

#### 5. Dockerfile Dependency Error
*   **Симптом**: Образ не збирався на Debian через відсутність пакета `libgl1-mesa-glx`.
*   **Причина**: У нових версіях Debian пакет перейменовано або розділено.
*   **Рішення**: Пакет замінено на `libgl1`.
    *   *Результат*: **Виправлено**.

### Моніторинг та поради:
1. **FFmpeg**: Якщо відео обробляється занадто довго — перевірте завантаження CPU. Ми використовуємо `-preset fast`.
2. **Пам'ять**: MediaPipe може споживати багато RAM. Якщо контейнер падає (OOM), збільште ліміти RAM на VPS.
3. **Локальне завантаження**: Ми додали `/upload-video` для файлів з ПК. Файли зберігаються в `downloads/`, які очищуються автоматично раз на 24 години.
